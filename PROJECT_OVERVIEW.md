# SmartCampus - Project Master Overview

This document serves as a comprehensive guide to the SmartCampus platform, explaining its mission, architecture, and technical inner workings.

---

## 1. Project Mission
SmartCampus is an intelligent career acceleration platform designed to bridge the gap between academic learning and industry placement. It provides students with data-driven guidance, skill gap analysis, and placement management tools.

## 2. Architecture Overview
The platform follows a **Client-Server Architecture**:
*   **Frontend (SmartCampusApp)**: A native Android application for Students, Officers, and Recruiters.
*   **Backend (SmartCampusBackend)**: A Kotlin Ktor server handling data, analytics, and external integrations.

---

## 3. Technology Stack

### Mobile (Android)
*   **Language**: Kotlin
*   **UI**: XML (Material Design)
*   **Networking**: Retrofit 2, Gson
*   **Maps**: WebView + Leaflet.js

### Backend
*   **Framework**: Ktor (Kotlin)
*   **Database ORM**: JetBrains Exposed
*   **Security**: JWT (JSON Web Tokens)
*   **Serialization**: Kotlinx Serialization
*   **External API**: GitHub API v3 (via Ktor Client)

## 4. Folder Structure Explained

### Core Backend Folders & Files
| Folder/File | Category | Meaning & Role |
| :--- | :--- | :--- |
| `src/main/kotlin` | **The Brain** | Contains 100% of the logic. If you want to change *how* the app works, look here. |
| `src/main/resources`| **The Settings** | Contains non-code assets like `application.conf` (DB passwords and server ports). |
| `Application.kt` | **The Entry Point** | The "Power Button" of your server. It's where the app starts up. |
| `SeedData.kt` | **The Librarian** | Populates the database with Admin accounts, Skills, and Sample Students on first run. |
| `SkillService.kt` | **The Skill Engine** | The most important logic file; calculates "Readiness %" and regional trends. |
| `GitHubTrendingService.kt` | **The Researcher** | The part of the system that "surfs" GitHub to see what's trending globally. |
| `application.conf` | **The Settings Book** | Stores database credentials, port numbers, and security keys. |
| `.gradle` & `gradle` | **Build System** | Automated folders that handle dependencies. **Do not edit manually.** |
| `build` & `bin` | **Temp Output** | Contains compiled code generated by the computer. These can be deleted safely. |

### SmartCampusApp (Android) Folders & Files
| Folder/File | Category | Meaning & Role |
| :--- | :--- | :--- |
| `app/src/main` | **The Core App** | Where the actual Android code (Kotlin), Layouts (XML), and Icons live. |
| `SmartCampusApi.java`| **The Phonebook** | A central list of every single URL the app can talk to on the server. |
| `build.gradle` | **The Recipe** | A list of all libraries (Retrofit, Gson, etc.) required to build the app. |
| `local.properties` | **Personal Path** | Tells the project exactly where the Android SDK is installed on *your* specific PC. |
| `settings.gradle` | **Project Index** | Tells the system which folders (like `app`) are part of this project. |
| `.idea` | **IDE Metadata** | Settings specific to Android Studio or IntelliJ. You don't need to touch this. |
| `gradlew.bat` | **The Starter** | A script used to build or run the app from the Windows command line. |
| `.gitignore` | **Filter** | Tells Git which files are private or temporary and should *not* be uploaded to GitHub. |
| `README.md` | **Intro** | The general "Welcome" document for the repository. |

## 5. Core "Smart" Logic

### GitHub Trend Analysis
The system identifies trending technologies by:
1.  Searching GitHub for repositories with >5,000 stars updated recently.
2.  Extracting languages and topics.
3.  Normalizing star counts into a **Demand Score (50% - 98%)**.
4.  Distributing global trends across 16 Indian tech hubs on the interactive map.

### Placement Readiness
Calculates a **Readiness %** by comparing a student's skills against the real-world demand found via the GitHub API and internal placement data.
